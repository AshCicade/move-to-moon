
## 消息确认

执行一个任务可能需要几秒钟的时间，您可能想知道，如果消费者启动一个长任务，并在完成之前终止，会发生什么。在我们当前的代码中，一旦RabbitMQ将消息传递给消费者，它会立即将其标记为删除。在这种情况下，如果终止一个worker，它正在处理的消息将丢失。发送给此特定worker但尚未处理的消息也会丢失。


如果RabbitMQ没有收到消费者的消息确认，会对这个消息进行重新排队。
